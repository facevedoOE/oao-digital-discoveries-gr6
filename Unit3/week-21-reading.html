Python Basics: Data Types & Variables

This week, you stop sketching algorithms and start writing real code. Python is the friendliest first programming language out there — and by the end of this lesson, you'll know how to store information in variables, recognize the four core data types, and chat with a user using print() and input().

Learning Objectives

By the end of this week, you will be able to:

1
Explain why Python is one of the most popular programming languages in the world — and where its goofy name actually comes from.
2
Create variables in Python using the = assignment operator and follow standard naming rules (lowercase_with_underscores).
3
Identify the four core data types — int, float, string, and bool — and pick the right one for a piece of real-world information.
4
Use print(), input(), and type() to display messages, ask the user questions, and inspect what kind of value Python is actually working with.
01
Section 1 · Why Python?

A friendly language that runs the world

Walk into the engineering building at Google, NASA, Netflix, Instagram, or Spotify and ask the people typing furiously at their keyboards what language they're writing in. A huge slice of them will answer the same thing: Python. It powers Instagram's back end. It steers some of NASA's rovers. Netflix uses it to figure out what show you'll binge next. Even the AI tools you've been hearing about — they're mostly trained and run with Python.

So why does a language that beginners learn in middle school also run billion-dollar companies? Because Python was designed with one rule baked in: code is read more often than it is written. The point of Python is to be easy on human eyes. Compare these two ways of asking the same question — printing the word Hello to the screen:

😵 Older Language (C)
Eight lines of ceremony
#include <stdio.h> int main() { printf("Hello\n"); return 0; }
🐍 Python
One line. Done.
print("Hello")

No header files, no main function, no semicolons. Just say what you want — Python figures out the rest.

That readability is no accident. Python was created in 1991 by a Dutch programmer named Guido van Rossum, who wanted a language that felt almost like English. He called it the language for everyone — kids, scientists, and grandmas alike — and over thirty years later, that promise still holds.

💡 Did You Know?

Python is not named after the snake. Guido van Rossum was a huge fan of the British comedy show Monty Python's Flying Circus, and he named his new language after the show because he wanted something "short, unique, and slightly mysterious." The official Python documentation is sprinkled with Monty Python jokes to this day — including references to "spam," "eggs," and the Spanish Inquisition (which nobody expects).

The snake-themed logos came later, after the name stuck. Now most Python tutorials use a snake mascot, but the language itself? Pure comedy nerd.

Here's the other reason Python runs the world: it's free. Anyone can download it. Anyone can use it. And there's a giant library of pre-written tools — millions of them — that other Python programmers have shared. Want to make a website? There's a Python tool for that. Want to make art with code? Yep. Want to teach a computer to recognize cats in photos? Three lines of Python and a lot of training data.

Throughout this unit, every example will be in Python. Don't worry — Python's whole job is to look obvious. Read along, and you'll be writing real code by the end of this lesson.

🌎 Real-World Connection

The Mars rovers Perseverance and Curiosity use Python in parts of their software. When NASA scientists in California want to plan tomorrow's drive across Mars — what to photograph, where to drill, how far to travel — they often write Python scripts to test the plan first.

The same language you're about to learn is being used to drive a robot on another planet. No pressure.

Section 2 · Variables — Labeled Boxes for Data

A variable is a name with a value attached

Imagine your room has a shelf full of clear plastic boxes. Each box can hold one thing — a sticker, a coin, a sandwich. To find anything later, you slap a sticky label on the front: TOOTHBRUSH, HOMEWORK, BIRTHDAY MONEY. The label is how you find the box. The contents can change at any time — you might spend the birthday money next week — but as long as the label says BIRTHDAY MONEY, you know which box to check.

That's exactly what a variable is in programming. A variable is a labeled box. The label is its name. Whatever's stored inside is its value. To put something into the box, you use the magic assignment operator — a single equals sign:

📝 Python — your first three variables # Each line creates a new variable player_name = "Riley" player_score = 0 is_game_over = False # Now let's look inside each box print(player_name) # → Riley print(player_score) # → 0 print(is_game_over) # → False

Read the = in your head not as "equals" but as "gets the value of" or "is set to." The line player_score = 0 says: "the box labeled player_score gets the value 0."

It's a one-way arrow. The name on the left gets the value on the right. Doing it backwards (0 = player_score) makes Python angry. The label has to be on the left.

You can also change what's inside a box later. The label stays. The contents update. That's why they're called variables — their values can vary over time:

🔁 Variables can change player_score = 0 # Start of game print(player_score) # → 0 player_score = 10 # Player scored! print(player_score) # → 10 player_score = player_score + 5 # Bonus points print(player_score) # → 15

Look closely at that last line: player_score = player_score + 5. To human eyes that looks confusing — how can a thing equal itself plus 5? But remember, the = isn't math. It's a one-way arrow. Python first calculates the right side (whatever's currently in the box, plus 5 → that's 15), and then stores that new total back into the box. The result: the score went up by 5.

Naming rules — the unwritten contract

Python doesn't care much what you name your variables, but other humans (including future-you) absolutely do. The whole Python community agrees on a style called snake_case: all lowercase, with underscores between words. Here's the contract:

✅ Good Names
player_name
high_score
is_logged_in
total_pizzas

Lowercase. Underscores between words. Describes the contents.

❌ Bad Names
x  — what does it hold?
PlayerName  — wrong case
2nd_score  — starts with digit
my variable  — spaces

Vague, mixed-case, or against Python's rules.

The hard rules — the ones Python won't even let you break:

  • A variable name cannot start with a number. player_1 is fine; 1_player is not.
  • No spaces ever. Use underscores.
  • You can use letters, digits, and underscores — nothing else (so no !, no $, no emoji).
  • Don't reuse reserved words like print, if, True. Python already uses those for itself.

And the soft rule — the one nobody enforces but everyone follows: names should describe the contents. Six months from now, when you reread your own code, x = 47 tells you nothing. students_in_class = 47 tells you everything.

Section 3 · The Four Core Data Types

Numbers, decimals, words, and yes-or-no

Every value in Python belongs to a data type. Why does that matter? Because Python treats different types differently. You can do arithmetic on numbers — but not on words. You can glue two strings together with + — but the result is the two strings stuck side-by-side, not added up. Knowing the type tells you what moves are legal.

For now, the four types you'll meet over and over again are:

🔢 Integers (int) — whole numbers

An integer — or int for short — is a whole number. No decimals. No fractions. It can be positive, negative, or zero. Anything you'd normally count with on your fingers (and a few your fingers run out for) is an integer.

🔢 Integers age = 12 high_score = 9001 temperature = -5 lives_remaining = 3

Use ints whenever you're counting things — players in a match, levels in a game, days until summer break.

📊 Floats (float) — numbers with a decimal point

A float is a number with a decimal in it — short for "floating-point number," because the decimal can float anywhere along the number. Use floats for measurements, money, or anything where halves and fractions matter.

📊 Floats height_meters = 1.52 pi = 3.14159 price = 9.99 two_point_zero = 2.0 # Still a float! The .0 makes it one.

Watch out: 2 is an int, but 2.0 is a float. The decimal point — even .0 — flips it.

📝 Strings (str) — text wrapped in quotes

A string — written str in Python — is a sequence of characters: letters, digits, spaces, punctuation, even emoji. The way Python knows you mean text and not code is the quotes. Single (') or double (") — Python is happy with either, as long as the opening and closing match.

📝 Strings greeting = "Hello, world!" favorite_food = 'pizza' empty = "" # Yes, an empty string is still a string. phone = "555-1234" # Numbers in quotes are strings, not ints!

That last line is a famous beginner trap: "555-1234" looks like a number, but the quotes make it a string. You can't do math on it.

✅ Booleans (bool) — yes or no, true or false

A boolean — bool for short — is the simplest data type of them all. It can hold exactly two values: True or False. That's it. Booleans show up everywhere, because programs are constantly making yes/no decisions: is the door locked? Is the player alive? Did the user click the button?

✅ Booleans is_logged_in = True game_over = False has_homework = True is_weekend = False

Capital T and capital F! true and false (lowercase) are not the same in Python — they'll trigger an error. And no quotes — "True" with quotes is a string, not a bool.

Section 4 · Talking with the User

print() shows things. input() listens.

A program that doesn't talk to the user is just doing math in a closet. To turn code into something a human can interact with, you need two friends from Python's standard toolkit: print() and input(). They're the mouth and ears of every Python program you'll write this year.

📢 print() — show something on the screen

You've seen print() already. You hand it something — a string, a number, a variable — and it displays it on the screen. That's the whole job.

📢 print() — output print("Welcome to Game Quest!") print(9001) print(True) # You can also print several things at once, # separated by commas — Python adds spaces for you. score = 15 print("Your score is", score, "points!") # → Your score is 15 points!

One of print()'s superpowers is that you can hand it variables instead of literal values, and it'll show whatever's currently inside. So your messages stay fresh — they always reflect the latest data.

🎤 input() — ask the user a question

Where print() talks to the user, input() listens for the user. Call input(), and Python pauses your program — the cursor blinks on the screen — until the human types something and presses Enter. Whatever they typed becomes a string, which you almost always want to capture into a variable.

🎤 input() — get a value from the user name = input("What's your name? ") print("Hi there,", name, "— welcome aboard!") # Sample run: # What's your name? Riley # Hi there, Riley — welcome aboard!

Two important truths about input():

  1. The thing in the parentheses is the prompt — the message Python shows the user before they type. Always end your prompt with a space, so the cursor doesn't sit right on top of the question mark.
  2. input() always returns a string. Even if the user types 12, what your program receives is the string "12". If you want a number, you have to convert it — and you'll learn how to do that in a couple of weeks.
💡 Did You Know?

The very first interactive Python program almost every learner writes — going back decades — is exactly the snippet above. It's so traditional that programmers have a name for any first program of any language: "Hello, World!" — coined in a 1972 Bell Labs textbook by Brian Kernighan, the same engineer who later co-wrote The C Programming Language.

Now you've written its descendant. Welcome to the club.

Here's what a five-line Python program looks like — using everything from this section. Read it slowly. You can already understand every line.

🧪 Mini-program: greet the user # Step 1 — say hello print("Welcome to Digital Discoveries!") # Step 2 — ask a question and store the answer favorite_color = input("What's your favorite color? ") # Step 3 — respond using the answer print("Nice choice —", favorite_color, "is a great color.")

Section 5 · The type() Detective

When you're not sure what kind of value you have, just ask

You'll meet code where you're not 100% sure what type a variable is. Maybe you wrote it last week. Maybe a friend handed it to you. Maybe it came from input(). Whatever the reason, Python has a built-in detective named type() that will tell you, on the spot, exactly what kind of value something is.

🔍 type() — Python's built-in detective print(type(42)) # → <class 'int'> print(type(3.14)) # → <class 'float'> print(type("hello")) # → <class 'str'> print(type(True)) # → <class 'bool'> # It works on variables too: score = 100 print(type(score)) # → <class 'int'>

That weird-looking output — <class 'int'> — is just Python's chatty way of saying "this belongs to the int family." The word inside the quotes ('int', 'float', 'str', 'bool') is the type's actual name.

The type() function is your debugging best friend. When something in your code isn't behaving right, ask Python to show you what type each value really is. The mystery usually solves itself.

🌎 Real-World Connection

This is the kind of trick that catches a real-world bug. A programmer at a music app once spent three hours wondering why their "Top 10 most-played songs" sort was wrong. Numbers like "7", "40", and "100" were sorting in the order "100", "40", "7" — the way you'd sort words in a dictionary.

The problem? Those weren't numbers. They were strings. The play counts had been read in from a file as text, and nobody noticed. One quick print(type(count)) would've found the bug in 10 seconds.

💡 Did You Know?

Python has way more than four data types. There are lists, tuples, dictionaries, sets, None, and a handful of others — and you can even make your own custom types with something called classes. But the four you learned today (int, float, str, bool) are the building blocks every other type is built on top of. You'll meet lists in Week 24, and the rest later in the course.

📚 Key Vocabulary — Week 21

Python
A beginner-friendly, general-purpose programming language created in 1991 by Guido van Rossum and named after Monty Python's Flying Circus.
Variable
A labeled box that stores a value. The label is the variable's name; the contents (the value) can change as the program runs.
Data type
The category a value belongs to — like int, float, string, or bool — which determines what operations are legal to do on it.
Integer (int)
A whole number with no decimal point. Examples: 0, 42, -7. Used for counting things.
Float
A number with a decimal point, like 3.14 or 2.0. Used for measurements, money, and fractions.
String (str)
A sequence of characters wrapped in single or double quotes — "hello" or 'pizza'. Used for text.
Boolean (bool)
A type that holds exactly one of two values: True or False. Used for yes/no decisions.
Assignment operator
The single equals sign =, which stores the value on its right into the variable on its left.
