Q1. Which BEST describes computational thinking?
───────────────────────────────────────────────────────────────
  A) Memorizing programming languages
     ❌ CT is a way of thinking, not memorization.
  B) A four-pillar approach: decompose, find patterns, abstract,
     write algorithms                                            ✓ CORRECT
     ✅ The four pillars work together on any problem.
  C) Building computer hardware
     ❌ Hardware is engineering. CT is problem-solving.
  D) Writing perfect code with no bugs
     ❌ Even great programmers write bugs. CT is about thinking
        clearly.

───────────────────────────────────────────────────────────────
Q2. Field-trip planning shows DECOMPOSITION when you...
───────────────────────────────────────────────────────────────
  A) Ask one teacher to handle every detail
     ❌ Delegation, not decomposition.
  B) Split into transportation, food, lodging, activities,
     safety                                                      ✓ CORRECT
     ✅ One big problem becomes five solvable ones.
  C) Brainstorm randomly with no structure
     ❌ No structure = no decomposition.
  D) Improvise on the day
     ❌ Improvising is the OPPOSITE of decomposition.

───────────────────────────────────────────────────────────────
Q3. Streaming-app "you might also like" — which CT pillar?
───────────────────────────────────────────────────────────────
  A) Decomposition
     ❌ Decomposition breaks problems into parts.
  B) Pattern recognition                                         ✓ CORRECT
     ✅ The app finds patterns in your viewing and matches them
        to other users' patterns.
  C) Abstraction
     ❌ Abstraction is hiding details.
  D) Algorithm finite-ness
     ❌ Finite-ness is a trait of algorithms, not a CT pillar.

───────────────────────────────────────────────────────────────
Q4. Which shape marks both START and END?
───────────────────────────────────────────────────────────────
  A) Diamond
     ❌ Diamonds are for yes/no decisions.
  B) Rectangle
     ❌ Rectangles are for process steps.
  C) Oval (terminator)                                           ✓ CORRECT
     ✅ The terminator is the only shape used for START and END.
  D) Parallelogram
     ❌ Parallelograms are for input/output.

───────────────────────────────────────────────────────────────
Q5. A correct decision diamond MUST have...
───────────────────────────────────────────────────────────────
  A) Exactly one outgoing arrow
     ❌ One arrow = no decision.
  B) Exactly two outgoing arrows, labeled YES and NO            ✓ CORRECT
     ✅ Diamonds split the path into two — that's their job.
  C) Three or more outgoing arrows
     ❌ Diamonds are yes/no — exactly two outcomes.
  D) No outgoing arrows
     ❌ The flow would dead-end.

───────────────────────────────────────────────────────────────
Q6. A flowchart and pseudocode for the same task...
───────────────────────────────────────────────────────────────
  A) Are completely different algorithms
     ❌ Same algorithm, different forms.
  B) Are two different ways to express the same algorithm        ✓ CORRECT
     ✅ Flowchart, pseudocode, and code = three forms of one
        algorithm.
  C) Cannot be translated between
     ❌ They CAN be translated.
  D) Mean different things to a computer
     ❌ Neither runs on a computer; only real code runs.

───────────────────────────────────────────────────────────────
Q7. Data type of `True` in Python?
───────────────────────────────────────────────────────────────
  A) Integer
     ❌ Integers are whole numbers.
  B) String
     ❌ Strings have quotes ("True" with quotes is a string).
  C) Boolean (bool)                                              ✓ CORRECT
     ✅ Booleans hold True or False — the only two values.
  D) Float
     ❌ Floats are decimal numbers.

───────────────────────────────────────────────────────────────
Q8. Best example of the DIGITAL DIVIDE?
───────────────────────────────────────────────────────────────
  A) Texting a friend
     ❌ Texting is using tech, not the divide.
  B) Some students having reliable home internet, others not    ✓ CORRECT
     ✅ The divide is unequal access to technology.
  C) Apple vs Android
     ❌ Platform choice, not access.
  D) New computer faster than old computer
     ❌ Speed differences ≠ digital divide.

───────────────────────────────────────────────────────────────
Q9. What does == do (different from =)?
───────────────────────────────────────────────────────────────
  A) Assigns a value to a variable
     ❌ A SINGLE = assigns. DOUBLE == compares.
  B) Compares two values for equality, returns True or False    ✓ CORRECT
     ✅ == asks "are these equal?" and returns True or False.
  C) Adds numbers together
     ❌ + adds.
  D) Divides numbers
     ❌ / divides.

───────────────────────────────────────────────────────────────
Q10. Pseudocode: age=14, IF age>=13 THEN PRINT "Welcome!" ELSE
     PRINT "Too young." — what prints?
───────────────────────────────────────────────────────────────
  A) "Too young."
     ❌ 14 IS >= 13, so the YES branch runs.
  B) "Welcome!"                                                  ✓ CORRECT
     ✅ 14 >= 13 is TRUE, so the IF branch prints "Welcome!".
  C) Both messages
     ❌ IF/ELSE only takes ONE branch.
  D) Nothing
     ❌ One of the two branches always runs.

───────────────────────────────────────────────────────────────
Q11. for i in range(5): — body runs how many times?
───────────────────────────────────────────────────────────────
  A) 4 times
     ❌ Off-by-one trap. range(5) = 0,1,2,3,4 = FIVE values.
  B) 5 times                                                     ✓ CORRECT
     ✅ range(5) gives 0,1,2,3,4 — exactly 5 iterations.
  C) 6 times
     ❌ range(5) STOPS before 5; it doesn't include 5.
  D) 0 times
     ❌ The loop runs at least once.

───────────────────────────────────────────────────────────────
Q12. TikTok endless feed — what core concept?
───────────────────────────────────────────────────────────────
  A) A loop running through a list of videos                     ✓ CORRECT
     ✅ Endless feeds are loops — for every user, every session,
        the app loops through millions of videos.
  B) A single function call
     ❌ A single call wouldn't keep showing more videos.
  C) A flowchart drawn by an artist
     ❌ Flowcharts are diagrams. The feed is code.
  D) A debugger checking each video
     ❌ Debuggers find bugs; they don't generate feeds.

───────────────────────────────────────────────────────────────
Q13. Which line correctly DEFINES a function in Python?
───────────────────────────────────────────────────────────────
  A) function add(a, b):
     ❌ Other languages use 'function'; Python uses 'def'.
  B) def add(a, b):                                              ✓ CORRECT
     ✅ Python: def + name + (params) + colon.
  C) define add(a, b)
     ❌ It's 'def', not 'define', and needs a colon.
  D) add(a, b) = function
     ❌ Not how any common language defines functions.

───────────────────────────────────────────────────────────────
Q14. In `def greet(name): print("Hi", name)` and `greet("Maya")`,
     what is `name` and what is `"Maya"`?
───────────────────────────────────────────────────────────────
  A) `name` is a parameter; `"Maya"` is an argument             ✓ CORRECT
     ✅ Placeholder in def = parameter. Value passed in = argument.
  B) Both are parameters
     ❌ Only the placeholder in def is a parameter.
  C) Both are arguments
     ❌ Only the passed-in value is an argument.
  D) `name` is the function call; `"Maya"` is the function name
     ❌ The function name is `greet`, not `name`.

───────────────────────────────────────────────────────────────
Q15. `if x = 5:` won't run — what KIND of bug?
───────────────────────────────────────────────────────────────
  A) Logic error — wrong math
     ❌ Logic errors run but get wrong answers. This refuses
        to run.
  B) Runtime error — happens only sometimes
     ❌ Runtime errors happen DURING execution. This is caught
        BEFORE running.
  C) Syntax error — line breaks Python's grammar rules           ✓ CORRECT
     ✅ Inside an IF, Python expects ==, not =. Grammar
        violation = syntax error, caught before the program
        runs.
  D) Network error — internet is down
     ❌ Bug is in the code, not the network.
