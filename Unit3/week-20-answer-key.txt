───────────────────────────────────────────────────────────────
Q1. Which flowchart shape marks the START or END of an algorithm?
───────────────────────────────────────────────────────────────
  A) Rectangle
     ❌ Rectangles are for process steps (calculations, actions).
        Not for start or end.
  B) Diamond
     ❌ Diamonds ask yes/no questions and split the path.
  C) Oval (terminator)                                            ✓ CORRECT
     ✅ The oval ("terminator") is the only shape used for START
        and END. Every flowchart begins and ends with one.
        (SC.K12.CTR.1.1)
  D) Parallelogram
     ❌ Parallelograms are for getting input or showing output —
        not for start or end.

───────────────────────────────────────────────────────────────
Q2. What is a DIAMOND shape used for in a flowchart?
───────────────────────────────────────────────────────────────
  A) Showing information to the user (output)
     ❌ That's the parallelogram's job (Input/Output shape).
  B) Marking the start of the program
     ❌ That's the oval's job (terminator).
  C) Connecting two arrows together
     ❌ Plain arrows connect shapes — diamonds are not connectors.
  D) Asking a yes/no question that splits the path                ✓ CORRECT
     ✅ Every diamond holds a yes/no question. It always has TWO
        arrows leaving — one for YES, one for NO.
        (SC.K12.CTR.2.1)

───────────────────────────────────────────────────────────────
Q3. How many arrows should leave a single DECISION DIAMOND?
───────────────────────────────────────────────────────────────
  A) Exactly two — one for YES, one for NO                        ✓ CORRECT
     ✅ A diamond ALWAYS has exactly two arrows leaving — labeled
        YES and NO. That's what makes it a "decision."
        (SC.K12.CTR.2.1)
  B) Exactly one — flowcharts always have a single path
     ❌ Flowcharts can branch — that's the whole point of a
        diamond. One outgoing arrow = no decision.
  C) Three or more, one for each possible answer
     ❌ A diamond is for yes/no questions only — exactly two
        outcomes.
  D) None — the diamond only receives arrows
     ❌ Diamonds receive AND send arrows. Without outgoing arrows
        the flow would dead-end at the question.

───────────────────────────────────────────────────────────────
Q4. Translating "IF temperature > 80 THEN PRINT 'It's hot!'" to
    a flowchart, how many shapes (besides terminators and arrows)?
───────────────────────────────────────────────────────────────
  A) Just one — only the IF
     ❌ The PRINT is its own step on the YES branch — needs its
        own shape.
  B) Two — a diamond for the IF and a parallelogram for the PRINT ✓ CORRECT
     ✅ One diamond for the IF condition ("temperature > 80?"),
        one parallelogram for PRINT (output). Each step needs its
        matching shape. (SC.K12.CTR.3.1)
  C) Three — two diamonds and one rectangle
     ❌ Only ONE decision here, not two. And PRINT is a
        parallelogram, not a rectangle.
  D) None — pseudocode and flowcharts can't be translated
     ❌ Pseudocode and flowcharts ARE translatable — they describe
        the same algorithm in different forms.

───────────────────────────────────────────────────────────────
Q5. Why does every flowchart need a TERMINATOR (oval) at the
    start AND at the end?
───────────────────────────────────────────────────────────────
  A) Because flowcharts crash without ovals
     ❌ Flowcharts don't "crash" — they're pictures, not running
        programs.
  B) Because Canvas requires it as a file format
     ❌ Canvas doesn't enforce flowchart shapes.
  C) Because terminators show where to begin reading the chart
     and where the algorithm ends                                 ✓ CORRECT
     ✅ Without START and END terminators, a reader has no idea
        where the algorithm begins or finishes. Terminators are
        the bookends of every flowchart. (SC.K12.CTR.1.1)
  D) Because diamonds need ovals to work properly
     ❌ Diamonds work fine on their own — they don't depend on
        ovals. Terminators serve a different role.
